Detailed Description

ArchiveInterface This is a base class that defines an interface for a archive oriented plugin A plugin like this is used to present the contents of an archive in the heirarchy view.

A plugin is accompanied by an cfg file which contains metadata about the plugin. This consists of supported file type, version, version control URL, and much more. See DataInterface

virtual QStringList ArchiveInterface::Directories ( )

Return a list of subdirectories in the archive. This gets called first when determining how to display an archive. Each subdirectory is "created" first. The subdirectories MUST be root through branch.
E.G. ["Folder 1", " Folder 2", "Folder 1/Sub Directory", "Folder 1/Sub Directory/Even Deeper"]

◆ Names()

virtual QStringList ArchiveInterface::Names ( )

Return a list of names for children in the archive as a path relative to the archive entry. This gets called second, and tells the program which directories to place the files inside of the archive.
E.G. ["Root.file", "Folder 1/File.sub", "Folder 1/File2.exe", "Folder 2/something.png"]
